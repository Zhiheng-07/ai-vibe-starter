# DESIGN.md — AI 简报站设计规范

> 气质:**一份暖灰色调、像印在纸上的 AI 读物。** 文字优先、克制、无彩色。
> 让读者一打开就想静下来读,而不是面对一个功能堆砌的软件界面。

---

## 0. 总原则(先读这条)

- **整个 UI 是暖灰阶,没有任何彩色。** 没有蓝色链接、没有绿色按钮、没有红色报错。唯一的"深"是近黑的墨色。
- **不用阴影、不用毛玻璃分层。** 表面之间只靠颜色深浅区分(奶油底 → 纸白卡片 → 墨黑按钮)。
- **卡片锐角(0 圆角),只有按钮和标签是全药丸形(9999px)。** 软按钮压在硬卡片上,是刻意的对比。
- **正文左对齐**,不居中(长文本居中难读)。
- 视觉语言是**文字优先**,不堆装饰。

---

## 1. 颜色 Tokens

| 名称                   | 值        | 用途                                           |
| ---------------------- | --------- | ---------------------------------------------- |
| 奶油底 Newsprint Cream | `#f7f4ed` | 页面背景、通栏底色——让整个界面像印刷纸而非屏幕 |
| 纸白 Paper White       | `#ffffff` | 卡片表面——靠色差从奶油底上"浮起来",不靠阴影    |
| 墨 Ink                 | `#242424` | 主文字、标题、正文(比纯黑稍暖)                 |
| 正文灰 Body Gray       | `#333333` | 次级正文                                       |
| 墨黑 Button Black      | `#191919` | 唯一的深色表面:药丸按钮背景                    |
| 边注灰 Marginalia Gray | `#6b6b6b` | 次要信息、分隔、日期、来源标签、弱化标注       |

**来源用官方 LOGO 标识,但一律单色化(灰阶),不保留品牌原色:**
| 来源 | 样式 |
|------|------|
| OpenAI | 官方 LOGO,渲染成单色墨黑 `#242424` |
| Anthropic | 官方 LOGO,渲染成单色墨黑 `#242424`(原橙色去掉) |
| 未知来源 | 用文字标签兜底(边注灰 `#6b6b6b`),无 LOGO 时不留空 |

> LOGO 单色化手段:优先用单色版 SVG;或对彩色 LOGO 加 CSS 滤镜(如 `filter: grayscale(1) brightness(0)`)统一压成墨黑。LOGO 高度建议 16–18px,和旁边日期文字对齐。
> ⚠️ 严守无彩色:Anthropic LOGO 的橙色必须去掉,不得作为页面彩色点缀。

## 2. 字体(全部用免费字体)

| 角色       | 字体(免费替代)                              | 用法                                             |
| ---------- | ------------------------------------------- | ------------------------------------------------ |
| 展示大标题 | **Playfair Display**(书卷气衬线,weight 400) | 页面顶部主标题"AI 简报"                          |
| UI 无衬线  | **Inter**                                   | 导航、按钮、副标题、来源标签、日期等所有 UI 文字 |
| 正文       | **系统衬线**(Georgia / Source Serif)        | 简报摘要正文,便于阅读                            |

- 中文回退:标题和正文都要带中文字体回退(如 `"Noto Serif SC"`、系统 `serif`),避免中文用到英文衬线字体导致难看。
- **签名规则:大标题用 weight 400,不加粗。** 靠字形本身的质感,不靠字重。

### 字号(已适配中文,大幅调小了原 120px)

| 角色                        | 字号    | 行高 | 字距         |
| --------------------------- | ------- | ---- | ------------ |
| 展示大标题(页面主标题)      | 40–48px | 1.1  | 略紧 -0.02em |
| 卡片标题(中文标题 titleZh)  | 20–22px | 1.4  | —            |
| 正文/摘要(summaryZh)        | 16px    | 1.6  | —            |
| 次要信息(日期/来源/caption) | 13px    | 1.4  | —            |

> ⚠️ 展示大标题为中文,尺寸须适配中文字形,不可过大导致爆屏。

---

## 3. 间距与形状

- **基础单位 8px**,密度舒适。用 8 / 16 / 24 / 48 / 64 这套尺度。
- **圆角**:卡片 0px、输入框 0px、**按钮 9999px、标签 9999px**。
- **页面最大宽度**:简报列表是单栏阅读,建议 **680–720px**(单栏读物,不是宽仪表盘)。
- 卡片内边距 16–24px;卡片之间间距 16–24px。

---

## 4. 版式(简报列表)

页面是一份**简报列表**——不是极简封面。版式从上到下:

```
┌─────────────────────────────────┐
│  Header:                        │
│  "AI 简报"(Playfair 大标题)     │
│  副标题一句 + 今日日期(边注灰)  │
├─────────────────────────────────┤
│  ★ 置顶高亮卡(最新且24h内)      │  ← 见 §5
├─────────────────────────────────┤
│  普通卡片                        │
│  普通卡片                        │  ← 按时间降序
│  普通卡片 …                      │
│  (默认最多 12 条)               │
├─────────────────────────────────┤
│  [ 查看全部 ] 药丸按钮           │  ← 超12条才出现,见 §5
├─────────────────────────────────┤
│  Footer:AI 免责声明 + 最后更新   │
└─────────────────────────────────┘
```

**单张卡片结构**(纸白底、锐角、无阴影):

- 顶部一行:来源 LOGO(单色墨黑,16–18px 高) + 日期(边注灰 13px)
- 中文标题 titleZh(20–22px)
- 中文摘要 summaryZh(16px,行高 1.6)
- 底部:"阅读原文 →"外链(墨色文字,hover 才出下划线,不用蓝色)

---

## 5. 展示分层(硬规则,必须照做)

1. **置顶高亮**:最新一条、**且发布在 24 小时内**的,置于列表最顶端并视觉突出。
   - 突出方式(用无彩色手段):更大的卡片 / 更粗的上边框 / 标题字号更大 / 一个"最新"小标记(灰阶)。**不用彩色高亮。**
   - 如果最新一条不在 24h 内,则不做置顶高亮,全部按时间降序平铺。
2. **其余按时间从新到旧排。**
3. **列表默认最多显示 12 条。**
4. **超过 12 条**:多余的折叠,页面底部出现一个 **"查看全部"药丸按钮**;点击后在本页展开全部剩余条目(不跳页)。

---

## 6. 组件规范

**药丸按钮(实心)** —— 用于"查看全部"

- 背景 `#191919`、白字、Inter 14px weight 400、内边距 8px×20px、圆角 9999px、无边框无阴影。

**来源标识** —— 见 §1:官方 LOGO 单色化墨黑,16–18px 高;无 LOGO 时用边注灰文字兜底。

**外链"阅读原文 →"**

- 墨色 `#242424` 文字,默认无下划线,hover 才出下划线。**不用蓝色。**

---

## 7. Do / Don't

**Do**

- 用 `#f7f4ed` 做所有通栏背景
- 卡片用 `#ffffff` 靠色差浮起
- 按钮/标签用 9999px 圆角,其余一律 0 圆角
- 靠颜色深浅分层,不用阴影
- 正文左对齐

**Don't**

- 不引入任何彩色 UI(无蓝链接、无绿按钮、无红报错)
- 不用阴影/毛玻璃分层
- 大标题不加粗(weight 400 是签名)
- 卡片/输入框不加圆角
- 正文不居中
- **不引付费字体**(GT Super / Söhne 等),只用上面列的免费字体

---

## 8. 给 UI 窗口的落地范围(边界)

- 只改 `app/` 下展示相关:`app/page.tsx`、`app/globals.css`、可新增展示用组件。
- **不碰 `lib/` 数据逻辑、不碰 `data/` 内容、不碰 main。**
- 字体用 Tailwind v4 `@theme` + Next 的字体方案(或 CSS `@import` 免费字体),确保中文有回退。
